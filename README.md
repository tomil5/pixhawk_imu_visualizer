# Pixhawk IMU — Инерциальная навигация в реальном времени

Программа считывает сырые данные IMU с полётного контроллера Pixhawk
по протоколу MAVLink и немедленно визуализирует движение в 3D —
ориентацию, траекторию от точки (0, 0, 0) и графики всех каналов.

---

## Запуск

```bash
# Права на порт (один раз после установки системы)
sudo usermod -a -G dialout $USER
# Перелогиниться, либо временно:
sudo chmod 666 /dev/ttyACM0

# Запуск (если не помогли права — через sudo)
sudo ./pixhawk_imu_reader
```

Окно визуализации открывается сразу при запуске — данные отображаются
в реальном времени по мере поступления пакетов с Pixhawk.
Нажмите **Ctrl+C** для завершения.

## Управление в окне

| Действие              | Эффект                              |
|-----------------------|-------------------------------------|
| ЛКМ + перетаскивание  | Вращение камеры                     |
| Колесо мыши           | Зум                                 |
| **R**                 | Сброс позиции и траектории в (0,0,0)|

Камера автоматически следит за полётным контроллером.

## Алгоритм инерциальной навигации

### 1. Калибровка (снятие bias)

Перед вычислениями из каждого измерения вычитается постоянное
смещение нуля датчика, определённое экспериментально по данным покоя:

```
a_x = (AX_raw − b_ax) · S_a       [м/с²],  S_a = 9.81 / 3050
ω_x = (GX_raw − b_gx) · S_g       [рад/с], S_g = 0.001

b_ax = 31,   b_ay = 24,   b_az = 0
b_gx = −7,   b_gy = 1.5,  b_gz = 3
```

### 2. Интегрирование кватерниона (ориентация)

Кинематическое уравнение связывает кватернион ориентации q
с угловой скоростью ω в системе координат тела:

```
dq/dt = 0.5 · q ⊗ Ω,    Ω = (0, ωx, ωy, ωz)
```

Для каждого шага Δt используется точная формула через ось-угол:

```
θ  = ‖ω‖ · Δt
dq = ( cos(θ/2),  ω/‖ω‖ · sin(θ/2) )
q  = q ⊗ dq  →  нормализация (борьба с накоплением ошибки)
```

### 3. Матрица поворота из кватерниона

Вектор из системы координат тела переводится в мировую систему
через матрицу вращения R, выраженную через компоненты кватерниона:

```
R₀₀ = 1−2(qy²+qz²)    R₀₁ = 2(qxqy−qwqz)    R₀₂ = 2(qxqz+qwqy)
R₁₀ = 2(qxqy+qwqz)    R₁₁ = 1−2(qx²+qz²)    R₁₂ = 2(qyqz−qwqx)
R₂₀ = 2(qxqz−qwqy)    R₂₁ = 2(qyqz+qwqx)    R₂₂ = 1−2(qx²+qy²)
```

### 4. Углы Эйлера (конвенция ZYX)

Из кватерниона вычисляются углы крена, тангажа и рыскания:

```
roll  (φ) = atan2( 2(qwqx + qyqz),  1 − 2(qx² + qy²) )
pitch (θ) = arcsin( 2(qwqy − qzqx) )
yaw   (ψ) = atan2( 2(qwqz + qxqy),  1 − 2(qy² + qz²) )
```

### 5. Двойное интегрирование (скорость и позиция)

Ускорение переводится в мировую СК и из него вычитается гравитация.
Последовательным интегрированием получаются скорость и позиция:

```
a_world = R · a_body − g_world      (g_world = [0, 0, −9.81] м/с²)

v(t+Δt) = v(t) + a_world · Δt      (1-й интеграл)
p(t+Δt) = p(t) + v(t)   · Δt      (2-й интеграл)
```

### 6. ZUPT — Zero Velocity Update

Ключевой метод борьбы с дрейфом позиции при отсутствии GPS.
Если норма угловой скорости ниже порога (контроллер покоится),
скорость мягко обнуляется, а малые ускорения считаются шумом:

```
если ‖ω‖ < 0.015 рад/с:
    v ← v · 0.85             (мягкое затухание)
    если |a_world| < 0.3 м/с²:  a_world ← 0
```

> **Примечание.** Двойное интегрирование без дополнительных сенсоров
> (GPS, барометр, оптический поток) неизбежно накапливает дрейф.
> Данный алгоритм предназначен для демонстрации принципа и
> корректно работает на промежутках 2-4 секунд активного движения.
